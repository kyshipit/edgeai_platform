# 适配器说明

> 代码均在 `runtime/adapters/`。平台总览见 [系统架构与运行逻辑.md](系统架构与运行逻辑.md)。

## 两类适配器

| 类型 | 目录 | 接口 | 说明 |
|------|------|------|------|
| **视觉槽** | `yolo/`、`scrfd/` | `IModelAdapter` | 每帧 `Preprocess→Inference→Postprocess`，由 `ModelCoordinator` 启停 |
| **逻辑旁路** | `llm/`、`tts/` | 无 `IModelAdapter` | 不进视觉 Pipeline；挂钩在 `platform/` 或 `LlmWorker` |

**修改检测逻辑时只改对应 `adapters/<name>/` 目录。**

---

## YOLOv5（`adapters/yolo/`）

| 文件 | 职责 |
|------|------|
| `yolo_postprocess.cpp` | 三尺度解码 `output[0..2]`，与正点原子 atk 例程逻辑对齐 |
| `yolo_adapter.cpp` | `init_yolov5_model`、RKNN 推理、`validate_yolov5_model_io` |

### 模型与路径

- 板端仅使用 **`./model/yolov5.rknn`**、**`./model/coco_80_labels_list.txt`**
- **不要**用仓库根目录 `deploy/` 产物替换
- **`./model/` 目录勿随意更换文件**

### 后处理要点

- `post_process`：固定解码 `output[0..2]`，要求 `n_output==3` 且 `dims[1]==255`（Init 会校验，不满足直接失败）
- **禁止**：9 路 box/conf/cls 分离解码、按 buffer 大小乱猜 output 索引

### 预处理

- OpenCV **BGR** 帧直接作为 `IMAGE_FORMAT_RGB888` 送 letterbox（与正点原子例程一致，**不要** BGR2RGB）

### 平台扩展

- `GetAdapterSignals()`：`person_present` → `ModelCoordinator` 切换 SCRFD

排障见 [YOLO与SCRFD问题排查记录.md](YOLO与SCRFD问题排查记录.md)。

## SCRFD（`adapters/scrfd/`）

| 文件 | 职责 |
|------|------|
| `scrfd_adapter.cpp` | `IModelAdapter` 实现；letterbox 640×640、RKNN 推理 |
| `scrfd_postprocess.cpp` | 三尺度人脸框 + 关键点解码（**9 路输出**布局） |

### 模型与路径

- 板端 **`./model/scrfd.rknn`**（`config/default.yaml` → `model.scrfd.path`）
- **勿**与 `yolov5.rknn` 混用；YOLO 为 3 路输出，SCRFD 为 9 路

### 启停

- 默认 idle 仅跑 YOLO；`person_present` 去抖稳定后由 `ModelCoordinator` 启用 SCRFD 槽
- 人脸信号供 `LlmGreeting` 门控；可选 `model.llm.preload_on_scrfd` 在 SCRFD 首次激活时预加载 LLM

排障见 [YOLO与SCRFD问题排查记录.md](YOLO与SCRFD问题排查记录.md)。

## LLM（`adapters/llm/`）

与 `adapters/yolo`、`adapters/scrfd` 同级，但 **不** 实现 `IModelAdapter` 每帧推理。

| 文件 | 职责 |
|------|------|
| `rkllm_session.*` | `rkllm_init` / **`rkllm_run`（同步）** / `rkllm_abort` / `rkllm_destroy`；回调 NORMAL 直写 stdout |
| `llm_worker.*` | 异步加载；**stat 预检**缺文件则 `Failed` 且跳过 `rkllm_init`；`infer_thread_` 跑 `RunPromptSync`；`IsLoadFailed()` |

第三方：`runtime/3rdparty/rkllm/`（`rkllm.h`、`librkllmrt.so`、`libgomp.so`）。**勿修改 3rdparty 目录内容。**

配置：`config/default.yaml` 中 `model.llm.*`。板端需自备 `.rkllm`。

**两条输出路径：**

- **用户对话**：`SubmitPrompt` → `rkllm_run` → 流式 `AI>`（`StaticCallback`）。
- **自动问候**：`LlmGreeting` 直接 `SetBannerLine(auto_greeting_text)`，**不经** RKLLM；**仅 `IsReady()`** 时输出（缺 `.rkllm` 时不问候）。

挂钩：`platform/llm_greeting.cpp` ← `ModelCoordinator::UpdateAfterFrame` ← 人脸信号。

集成行为、门控与终端 UX 详见 [LLM与ModelCoordinator集成.md](LLM与ModelCoordinator集成.md)。

## TTS / MeloTTS（`adapters/tts/`）

### 作用

将终端 **`AI>` 助手话术** 合成为语音（44100Hz），包括：

- 静态自动问候（`SetBannerLine`，可 `skip_static_greeting` 关闭播报）
- `YOU>` 后 RKLLM 整轮回复（`reply_accumulator` 与 stdout 同源）

### 模型与配置

板端 `./model/`：

- `encoder-ZH_MIX_EN.rknn`、`decoder-ZH_MIX_EN.rknn`
- `lexicon.txt`、`tokens.txt`

YAML：`config/default.yaml` → `model.llm.tts.*`（需 `model.llm.enabled: true`）。

### 播放

合成后写 `/tmp/edgeai_tts.wav`，程序内由 `AudioPlayer` 调用：

```bash
gst-play-1.0 -q --no-interactive /tmp/edgeai_tts.wav
```

板端需已安装 `gst-play-1.0`。

模块表、数据流与验收见 [TTS与MeloTTS集成说明.md](TTS与MeloTTS集成说明.md)。
