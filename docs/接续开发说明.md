# EdgeAI Runtime 接续开发说明

> 新对话请 **@本文件**。平台总览见 [系统架构与运行逻辑.md](系统架构与运行逻辑.md)。规划新模型见 [模型演进与待办.md](模型演进与待办.md)。排障见 [YOLO与SCRFD问题排查记录.md](YOLO与SCRFD问题排查记录.md)。

## 0. 必读

1. 板端只认 `runtime/` + `./model/*.rknn`；`deploy/` 不参与板端运行。
2. YOLO 须 `output num: 3`；9 路输出多为 yolo 误加载 scrfd。见排查专文。
3. 依赖 `yolov5.rknn`、`scrfd.rknn`；可选 `.rkllm`、MeloTTS 权重。缺 `.rkllm` 时**仅视觉降级**（见 [LLM与ModelCoordinator集成.md](LLM与ModelCoordinator集成.md) §5–§6）。
4. 崩溃/Ctrl+C： [错误修复调试说明.md](错误修复调试说明.md)。

## 1. 分层


| 目录                             | 职责                           |
| ------------------------------ | ---------------------------- |
| app                            | 配置与启动                        |
| engine                         | Pipeline、队列                  |
| platform                       | ModelCoordinator、LlmGreeting |
| adapters/yolo, scrfd, llm, tts | 模型插件                         |
| display, capture               | 显示与采集                        |


## 2. 多槽


| 场景     | 槽                                                        |
| ------ | -------------------------------------------------------- |
| idle   | yolo                                                     |
| person | yolo + scrfd                                             |
| 问候     | llm 逻辑槽：静态文案 `SetBannerLine`（**须模型 Ready**）；用户对话 `SubmitPrompt` → RKLLM |
| 语音     | tts 逻辑槽：播 `AI>` 助手话术（见 default.yaml `model.llm.tts`）     |


## 3. 配置

见 `runtime/config/default.yaml`：`model.yolo`、`model.scrfd`、`model.llm`、`system.switch.*`。

## 4. 编译

```bash
cd runtime && ./build-linux.sh
./edgeai_platform_app config/default.yaml
```
